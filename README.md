## 一、设计模式

设计模式遵循六⼤大原则：单⼀一职责( ⼀一个类和⽅方法只做⼀一件事 )、⾥里里⽒氏替换( 多态，⼦子类可扩展⽗父类 )、依赖倒置( 细节依赖抽象，下层依赖上层 )、接⼝口隔离( 建⽴立单⼀一接⼝口 )、迪⽶米特原则( 最少知道，降低耦合 )、开闭原则( 抽象架构，扩展实现 )。
